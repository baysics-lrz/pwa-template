REST Web-Services

Overview

Welcome to the documentation of the portal's REST API!
The API can be used after successful registration and Token-Authentification from any program or programming language. For beginners we recommend here the programming language Python with library requests or the program Postman.

Requests
  • Filtering parameters for Information GET
    Filtering parameters are passed to the "GET" interfaces as URL parameters
  • Other parameters
    Requests like "ADD" or "UPDATE" must be passed in JSON or 'form-data' format. Note that when uploading a photo file, the 'form-data' format must be used, since files in JSON format cannot be transferred.

Responses
Response parameters are given in JSON format only.

Login

Interface: http://127.0.0.1:8000/api/accounts/login
Request: POST
Format (JSON or form-data):
  • username:
    • your username
    • Data type: String
  • password:
    • your password
    • Data type: String
Response (upon successful registration): HTTP/1.1 200 OK
  • token:
    • A token string with which the user can authenticate
Response (for incorrect request): HTTP/1.1 400 Bad Request
Example:
  • Request:
    POST http://127.0.0.1:8000/api/accounts/login HTTP/1.1 
                        {
                        "username": "your username",
                        "password": "your password"
                    }
  • Response:
    HTTP/1.1 200 OK 
                            {
                            "token": "0e434ffb8fd31a0b787f85d607547f5994e0a5f3",
                        }

Information:
The obtained token 0e434ffb8fd31a0b787f85d607547f5994e0a5f3 can now be used to identify yourself to all interfaces. For this, you have to add the field authentification "Token 0e434ffb8fd31a0b787f85d607547f5994e0a5f3".

User information

Interface to get information about the ID, user name and e-mail of the logged-in user.
Interface: http://127.0.0.1:8000/api/accounts/user
Request: POST
Header:
  • Authentification:
    • The authentification-token of the user
    • Format: Token {token_string}
Response:
Response (with correct authorization): HTTP/1.1 200 OK
  • id:
    • The User-ID of the user
  • username:
    • The username of the user
  • email:
    • The email-address of the user
Response (in case of incorrect authentication): HTTP/1.1 403 Forbidden



Category1

Parameter

  • id (Read-Only):
    • The unique identification number of the category1 entry
    • Data type: Number
  • Category1Subject:
    • The category1 subject
    • Data type: String
  • Certainty:
    • Statement about whether the observer is certain or uncertain about the observation
    • Data type: String
    • Value: "Certain" or "Uncertain"
  • Number:
    • dummy text
    • Data type: Number
    • Value: A positive number (above 0)
  • Category1Feature1:
    • dummy text
    • Data type: String
    • selection:
  • Category1Feature2:
    • dummy text
    • Data type: String
  • Category1Feature3:
    • dummy text:
    • Data type: String
    • selection:
  • Lat:
    • Latitude of the observation
    • Data type: String
  • Lon:
    • Longitude of the observation
    • Data type: String
  • Municipal (Read-Only):
    • The municipal in which the observation is made
    • Data type: String
  • ObservationDate:
    • The date of the observation
    • Data type: String
    • Format: "YYYY-MM-DD"
  • ObservationTime:
    • The date of the observation
    • Data type: String
    • Format: "HH-MM-SS"
  • Photo:
    • The URL to the corresponding photo
    • The file (when uploading) or URL-string of the uploaded photo

Request: GET
Header:
  • Authentification:
    • The authentification-token of the user
    • Format: Token {token_string}
Filter-Parameter: category1-Parameter
Response (with correct parameters): HTTP/1.1 200 OK
Response (with wrong authentification): HTTP/1.1 403 Forbidden

Interface: http://127.0.0.1:8000/api/observations/category1/add
Request: POST
Header:
  • Authentification:
    • The authentification-token of the user
    • Format: Token {token_string}
Parameter (JSON or form-data): category1-Parameter
Response (with correct parameters): HTTP/1.1 201 Created
Response (with wrong authentification): HTTP/1.1 403 Forbidden
Response (with incorrect parameters): HTTP/1.1 400 Bad Request
