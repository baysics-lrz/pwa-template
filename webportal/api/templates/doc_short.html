Attribute of observations

Attributes of Category1 observation:
  • id:
    • The unique identification number of the category1 entry
    • Data type: Number
  • Category1Subject:
    • The Category1 subject
    • Data type: String
  • Certainty:
    • Statement about whether the observer is certain or uncertain about the observation
    • Data type: String
    • Value: "Certain" oder "Uncertain"
  • Number:
    • The number of observed category1
    • Data type: Number
    • Value: Positive number (above 0)
  • Category1Feature1:
    • dummy text
    • Data type: String
    • Selection:
  • Category1Feature2:
    • dummy text
    • Data type: String
  • Category1Feature3:
    • dummy text
    • Data type: String
    • Selection:
  • Lat:
    • Latitude of the observation
    • Data type: String
  • Lon:
    • Longitude of the observation
    • Data type: String
  • Municipal:
    • The municipal in which the observation is located
    • Data type: String
  • ObservationDate:
    • Date of the observation
    • Data type: String
    • Format: "YYYY-MM-DD"
  • ObservationTime:
    • The time of the observation
    • Data type: String
    • Format: "HH-MM-SS"
  • Photo:
    • The URL to the corresponding photo
    • Data type: String

Interfaces for information queries

The interfaces for information queries provide a service that makes it possible to obtain information about all observations in a clearly arranged JSON format.
The response of the API interfaces is always a list of JSON objects containing all attributes of the observations. Certain queries can also be filtered using query parameters.
Category1
  • Method: GET
  • API: http://127.0.0.1:8000/api/observations/category1
Category2
  • Method: GET
  • API: http://127.0.0.1:8000/api/observations/category2

Category3
  • Method: GET
  • API: http://127.0.0.1:8000//api/observations/category3

Category4
  • Method: GET
  • API: http://127.0.0.1:8000/api/observations/category4
